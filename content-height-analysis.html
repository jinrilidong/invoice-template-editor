内容区域高度分析

📐 内容区域总高度
内容区域容器高度: 723px
🔍 各个Section的高度分析

1️⃣ Header Section

Header容器高度 (h-12): 48px (3rem = 48px)

2️⃣ Info Section

Info项目高度 (每个item): 20px (估算值)
Info项目数量: 动态

3️⃣ Table Section

Section标题高度 (h-2.25): 9px (0.5625rem = 9px)
Subsection标题高度 (h-2.25): 9px (0.5625rem = 9px)
Table表头高度 (h-3.25): 13px (0.8125rem = 13px)
Table行高度 (h-3.25): 13px (0.8125rem = 13px)
Summary区域高度 (h-4.25): 17px (1.0625rem = 17px)

4️⃣ Description Section

Description内容高度: 动态 (约20px per 100字符)

5️⃣ Item Section

Item项目高度 (每个item): 20px (估算值)
Item项目数量: 动态
📊 高度计算示例

假设所有section都显示的情况：

  • Header: 48px
  • Info (3个items): 3 × 20px = 60px
  • Table (1个table, 5行): 9px + 9px + 13px + (5 × 13px) + 17px = 113px
  • Description (200字符): 40px
  • Item (2个items): 2 × 20px = 40px
总内容高度: 48 + 60 + 113 + 40 + 40 = 301px
🎯 关键发现

内容区域高度分配：

  • 内容区域总高度: 723px
  • 实际内容高度: 约300-400px (取决于内容数量)
  • 剩余空间: 约300-400px
  • gap间距: 10px (各section之间)

高度限制：

  • 当内容超过723px时，会触发分页
  • 分页计算使用maxHeightPerPage = 723px
  • Footer区域(9px)在内容区域外部
