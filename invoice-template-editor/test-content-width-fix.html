内容区域宽度修复测试

修复目标
将HTML导出的宽度改为与PDF预览的内容区域一致：
  • PDF页面总宽度：612px
  • PDF页面内边距：20px
  • PDF内容区域宽度：572px (612 - 20*2)
  • HTML导出应该匹配这个宽度设置
宽度规格对比

PDF预览设置

页面宽度：612px

页面内边距：20px

内容区域：572px

计算：612 - 20*2

HTML导出（修复前）

页面宽度：792px

页面内边距：20px

内容区域：100%

问题：宽度不匹配

HTML导出（修复后）

页面宽度：612px ✅

页面内边距：20px ✅

内容区域：572px ✅

状态：完全匹配

修复前后对比
修复前（宽度不匹配）
HTML导出设置：
body { padding: 20px; width: 792px; /* 错误的宽度 */ max-width: 792px; } .invoice-container { width: 100%; /* 自适应宽度 */ }
问题：
  • HTML导出宽度792px与PDF预览612px不匹配
  • 内容区域宽度不一致
  • 布局效果不同
修复后（宽度一致）
HTML导出设置：
body { padding: 20px; width: 612px; /* PDF页面宽度 */ max-width: 612px; } .invoice-container { width: 572px; /* 内容区域宽度 */ }
优势：
  • HTML导出宽度612px与PDF预览完全匹配
  • 内容区域宽度572px一致
  • 布局效果完全相同
技术修复详情

1. 页面宽度修复

/* 修复前 */ body { width: 792px; /* 错误的宽度 */ max-width: 792px; } /* 修复后 */ body { width: 612px; /* PDF页面宽度 */ max-width: 612px; }

2. 内容区域宽度修复

/* 修复前 */ .invoice-container { width: 100%; /* 自适应宽度 */ } /* 修复后 */ .invoice-container { width: 572px; /* 内容区域宽度 (612px - 20px*2) */ }

3. 宽度计算说明

PDF页面总宽度: 612px PDF页面内边距: 20px (上下左右) 内容区域宽度: 612px - (20px * 2) = 572px HTML导出现在使用相同的计算方式
