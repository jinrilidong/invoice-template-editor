Header上边距问题分析

问题描述

🐛 问题现象：

Header设置了24px的上边距，但渲染的PDF看起来超过了24px。

🔍 可能原因分析：

  • PDF预览vs HTML导出差异：PDF预览使用flex gap，HTML导出使用margin
  • 累积间距：可能存在多个间距叠加
  • 容器间距：父容器的间距影响
  • 字体行高影响：字体大小和行高可能影响视觉间距
PDF预览 vs HTML导出对比

PDF预览设置

容器：flex布局

间距：gap: 10px

Header：无额外margin

实际间距：10px

HTML导出设置

容器：table布局

间距：section-spacer: 8px

Header：margin-top: 24px

实际间距：24px

问题分析

差异：14px (24px - 10px)

原因：PDF预览用gap，HTML用margin

影响：视觉不一致

需要：统一间距设置

间距累积分析
当前HTML导出（问题）
/* 可能的间距累积 */ body { padding: 20px; /* 页面内边距 */ } .header-table { margin-top: 24px; /* Header上边距 */ } /* 总间距 = 20px + 24px = 44px */
问题：
  • 页面内边距20px + Header边距24px = 44px
  • 与PDF预览的10px差距很大
  • 视觉上看起来间距过大
PDF预览（参考）
/* PDF预览的间距设置 */ .pdf-content { gap: 10px; /* 统一的section间距 */ } .header-section { /* 无额外margin */ } /* 总间距 = 页面内边距 + gap = 20px + 10px = 30px */
优势：
  • 使用统一的gap间距
  • 所有section间距一致
  • 视觉上更协调

解决方案

方案1：统一使用gap间距（推荐）

/* 修改HTML导出，使用统一的section间距 */ .header-table { margin-top: 10px; /* 改为10px，与PDF预览一致 */ } /* 或者使用section-spacer */ .section-spacer { height: 10px; /* 改为10px */ }

方案2：调整页面内边距

/* 减少页面内边距 */ body { padding: 10px; /* 从20px改为10px */ } .header-table { margin-top: 24px; /* 保持24px */ }

方案3：使用相对单位

/* 使用相对单位，与PDF预览保持一致 */ .header-table { margin-top: 10px; /* 与PDF预览的gap一致 */ }
推荐修复方案

最佳解决方案：统一间距为10px

/* 修改HTML导出中的header间距 */ .header-table { width: 100%; border-collapse: collapse; margin-top: 10px; /* 从24px改为10px */ } /* 同时调整section-spacer */ .section-spacer { height: 10px; /* 从8px改为10px */ }
优势：
  • 与PDF预览完全一致
  • 所有section使用统一的10px间距
  • 视觉上更协调
  • 符合设计规范
