PDF渲染修复测试

问题描述

🐛 问题现象：

  • PDF渲染后没有换行：长文本在PDF中不换行，导致内容溢出
  • HTML导出宽度自适应：导出的HTML宽度自适应，应该限制为792px

🔍 问题原因：

  • 缺少CSS换行属性：`word-wrap`、`overflow-wrap`、`white-space`
  • HTML导出没有固定宽度限制
  • 表格单元格没有换行支持
修复方案对比

1. 宽度限制修复

/* 修复前 */ body { padding: 20px; /* 缺少宽度限制 */ } /* 修复后 */ body { padding: 20px; width: 792px; /* 固定宽度792px */ max-width: 792px; /* 限制最大宽度 */ }

2. 换行支持修复

/* 修复前 */ .data-table td { padding: 2px 0; font-size: 7px; /* 缺少换行属性 */ } /* 修复后 */ .data-table td { padding: 2px 0; font-size: 7px; word-wrap: break-word; overflow-wrap: break-word; white-space: normal; }

技术修复详情

1. 宽度限制修复

body { margin: 0; padding: 20px; font-family: 'Inter', -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif; font-size: 12px; line-height: 1.2; min-height: 752px; width: 792px; /* 固定宽度792px */ max-width: 792px; /* 限制最大宽度 */ }

2. 换行支持修复

/* 为所有文本元素添加换行支持 */ .data-table th, .data-table td { word-wrap: break-word; /* 允许长单词换行 */ overflow-wrap: break-word; /* 现代浏览器的换行支持 */ white-space: normal; /* 确保文本可以换行 */ } .info-item-label, .info-item-value, .table-section-title, .table-subsection-title, .subtotal-value, .summary-content, .description-label, .description-text, .item-section-title, .item-item-label, .item-item-value, .footer-section td { word-wrap: break-word; overflow-wrap: break-word; white-space: normal; }
