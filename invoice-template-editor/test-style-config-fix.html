🔧 PDF Style Config 修复测试
此测试用于验证修复后的 PdfCompatibleRenderer 组件是否正确响应 styleConfig 参数的变化。
测试步骤：
  1. 打开应用：http://localhost:5173
  2. 切换到 "PDF Style" 标签页
  3. 调整以下参数并观察预览变化：
    • Base Font Size: 从 7px 改为 10px
    • Section Gap: 从 10px 改为 20px
    • Header Title Size: 从 20px 改为 24px
    • Header Title Color: 改为红色 #ff0000
  4. 检查预览是否实时更新
  5. 导出HTML并检查是否包含新的样式值
预期结果：
✅ 调整 Base Font Size 时，整个PDF的字体大小应该改变
✅ 调整 Section Gap 时，section之间的间距应该改变
✅ 调整 Header Title Size 时，标题字体大小应该改变
✅ 调整 Header Title Color 时，标题颜色应该改变
✅ 所有变化都应该实时反映在预览中
✅ 导出的HTML应该包含新的样式值
🐛 修复的问题
本次修复解决了以下关键问题：
修复内容：
  1. Props 解构问题: 修复了 defineProps 没有正确解构的问题
  2. 硬编码样式: 将硬编码的 fontSize: '7px' 改为使用 styleConfig.baseFontSize
  3. Section间距: 将硬编码的 height: 10px 改为使用 styleConfig.sectionGap
  4. Props引用: 所有组件引用都添加了 props. 前缀
📋 技术细节
修复的具体代码变更：
关键修复：
  1. Props 解构: 
    // 修复前
defineProps<{...}>()

// 修复后  
const props = defineProps<{...}>()
  2. 动态样式: 
    // 修复前
fontSize: '7px'

// 修复后
fontSize: (props.styleConfig?.baseFontSize || 7) + 'px'
  3. 动态间距: 
    // 修复前
style="height: 10px; padding: 0"

// 修复后
:style="{ height: (props.styleConfig?.sectionGap || 10) + 'px', padding: 0 }"
